Stage 1: Enterprise Data Cleaning

Implemented a robust pre-processing layer that handles missingness (e.g., tenure-based nulls in TotalCharges) and ensures data type consistency for heavy numerical analysis.

Stage 2: Strategic Feature Engineering

Created domain-specific features to enhance AI accuracy:

  • Tenure Binning: Capturing non-linear decay of customer loyalty.
  • Revenue Segmentation: Applying quantitative grouping for financial impact analysis.
  • CLV Calculation: Weighting risk by financial value.
Stage 3: Advanced SQL Normalization

Managed 3NF schema design across 4 primary tables to demonstrate relational proficiency and prevent data redundancy.

Machine Learning Model: Logical Flow

The system utilizes a dual-model approach with Random Forest for predictive power and Logistic Regression for business interpretability.

Software Engineering Best Practices
  • Modularity: Decoupled logic across specialized scripts (EDA, ML, ETL).
  • Automated Orchestration: Single entry point (`run_pipeline.py`) for end-to-end execution.
  • Environment Integrity: Managed dependencies via `.venv` and `requirements.txt`.
